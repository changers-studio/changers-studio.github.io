How It Works

Workflow Overview for Secure and Efficient AI Driven Blockchain Transactions

AI Agent Proposes a Transaction

When a user instructs an AI agent (e.g., “transfer 1 ETH to Contract X” or “deploy a new ERC-20 token”), the agent bundles three items:

  • Transaction Payload
  • Original Prompt
  • Text Result
Simulation

Before executing any on-chain transaction, OPUS Layer simulates it in a secure, isolated environment. This process identifies risks like malicious contract interactions, unexpected token behavior, and phishing attempts. By testing each step before it reaches the blockchain, OPUS Layer ensures transaction integrity, safeguarding both user assets and the AI agent.

LLM Cross-Check

Once the transaction is deemed safe, OPUS Layer ensures it aligns with the user’s original instructions. A specialized LLM verifies the transaction details against the prompt, and hardcoded limits prevent exceeding predefined thresholds. By combining advanced LLM checks with deterministic guardrails, OPUS Layer ensures every transaction follows the user’s intent within a controlled, secure environment.

On-Chain Confirmation

Once the transaction passes both simulation and verification, OPUS Layer produces a tamper-proof cryptographic report—a verifiable “receipt” detailing the AI agent’s intended actions and confirming compliance with all safety and instruction checks. This report is then submitted on-chain with the final transaction, ensuring a transparent, immutable record of the verification process. By providing a trusted audit trail, OPUS Layer enables users, developers, and auditors to easily review and confirm that each transaction is executed correctly and securely.
